## ⚠️ ヒロに確認したいポイント(Phase 5前に判断)

### 1. ブース構造の解釈(最重要)

ヒアリングメモ D「同日付の連続**行** = ブース順」と実シート構造が食い違う。
実シートを見ると、機械ブロックは**列方向**に展開されている(機械名の右に同じヘッダパターンが繰り返される)。

例: ファミマ呉服町
  - R1 col 4='カリーノミニSP'(以降 col 18, 32 は機械名空) → カリーノミニSP×3ブース
  - R1 col 46='ロイヤルビンゴ' → ロイヤルビンゴ×1ブース
  - R1 col 60='カリーノ７' col 74 空 → カリーノ７×2ブース

**parserは「列方向ブース」で実装した**。これで合っているか? 違う場合はロジック修正が必要。

### 2. 同日付に複数行(同一機械・同ブース)

ファミマ呉服町 sheet では カリーノミニSP ブース1 で 4/28日付の行が4つ続く(景品名・メーター値はそれぞれ異なる)。
これは「同日の複数巡回 or 景品入れ替えイベント」を別行で記録している? それとも記入ミス?

### 3. 福重シートの集計列(合計/一階/二階吹抜 等)

R2 col 0-6 に「日付/合計/一階/二階吹抜/二階奥/二階キッズ/二階SR」のサマリ列があり、その先(col 8 以降)に機械1のヘッダがある。
parserは col 0 のブロック範囲を「機械1ブロック」として扱っている(機械名: 福重BUZZミニ⑩ | 1F)。
この場合 col 8(100円) col 10(プライズ) を機械1の値として取り込んでいるが、これでOK?

### 4. R番号(DFX所有)検出

R番号付き機械(R1014 / R3003 / R3004 等)を上記サマリで一覧化。これらは DFX 所有としてマークすべき機械。
上記リストに抜け漏れ・誤検出ないか確認。
