## 提案: 異常機械の扱い(ヒロ判断要)

**選択肢A**: max_booth > 4 の行は本投入から **除外**(安全側)
  - メリット: ヒロ仕様『1/2/4ブース』厳守、後で機械マスタとの照合楽
  - デメリット: 過去の8ブース時代データ等が失われる

**選択肢B**: max_booth > 4 の行は **そのまま投入**、Dashboard側で別途filter
  - メリット: 元データ完全保存
  - デメリット: Round Zero Dashboardでブース指標が歪む可能性

**選択肢C(私の推奨)**: b1〜b4 は本投入 + b5以降は **別sourceで分離保存**(`import_fukuoka_2026_v2_anomaly`)
  - メリット: 本指標は綺麗、anomalyは後で精査可能
  - デメリット: 投入処理が少し複雑
