ClawOps v2 リデザイン案

現状の問題点
使いづらさの原因を分類

🔴 1. 巡回入力フロー — 画面遷移が多すぎる

ホーム → 店舗選択 → 機械リスト → ブース入力 → 下書き確認 → 完了
6画面も遷移が必要。毎回の巡回で繰り返すのはストレス。

→ v2案: 店舗チップ＋機械タブで1画面に統合。入力→保存を2タップで完結。

🔴 2. BoothInput — フィールドが多すぎ＆小さすぎ

1ブースに10個の入力欄（IN/OUT/残/補/A/C/L/R/O/景品名）。
設定値A〜Oは毎回変えるものではないのに常時表示。
入力欄36px幅では指で正確にタップできない。

→ v2案: IN/OUTを大きく（18px font）。設定A〜Oは折りたたみ（変更時のみ展開）。
→ 景品名は前回値を自動表示、変更時のみ編集。

🔴 3. 画面の見た目 — 情報密度が高すぎる

ダークテーマ自体は良いが、余白が少なく文字が小さい。
10px/11pxの文字、48px幅のinputが横に7個並ぶのは視認性が悪い。
ブースカード間のスペースも1.5しかなく、どこからどこまでが1ブースか分かりにくい。

→ v2案: カード間のスペースを広げ、セクションを明確に分割。
→ メーター入力は最低14px以上、タップターゲットは44px以上確保。

🟡 4. ナビゲーション — 今どこにいるか分からない

←ボタンで戻るだけのナビゲーション。
現在の店舗名・機械名がヘッダーに小さく表示されるだけで、
全体の中の現在地が把握しづらい。

→ v2案: 下部タブバーで「入力/ダッシュボード/管理」を常時アクセス可能に。
→ 店舗は上部チップで固定表示＆切替可能。

🟡 5. データ確認 — 売上をすぐ見れない

ランキング画面は別ルートで遷移が必要。
入力中に「この機械の売上どうだっけ？」が確認できない。

→ v2案: ダッシュボードタブで売上・ランキングを即確認。
→ 入力画面でもブースごとに売上差分をリアルタイム表示。

🟡 6. 管理画面 — ホーム画面がごちゃごちゃ

StoreSelectにモード切替・QRスキャン・管理メニュー3つ・店舗リストが全部詰め込まれている。
日常の入力作業と管理作業が混在。

→ v2案: 管理機能は専用タブに分離。入力タブはひたすら入力に集中。
結論: 部分修正ではなくUI構造の再設計が必要
ルーティング構造・ページ配置・コンポーネント設計を
業務アプリ型（タブナビ＋フラット構造）に移行。
sheets.jsのデータ層はそのまま活用。
